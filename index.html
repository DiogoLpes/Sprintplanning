📋 Como acompanhar este Sprint no dia a dia

Guia de operações para garantir o sucesso do Sprint 1 — Projeto Integra-te

1 Sprint Kickoff (Dia 1)
  • →Reunião de 30min: rever este board em conjunto e confirmar quem pega em quê.
  • →Adrien: arrancar CI-01 (GitHub Actions) — desbloqueia PRs para todos.
  • →Adrien + Diogo: começar BE-01 (modelos) — o frontend depende da API.
  • →Giulio: começar FE-01 (Tailwind config + Router) — pode avançar em paralelo.
  • →Cristian: ler o código base e fazer setup local. Só começa CRS-01 depois de FE-02 estar pronto.
2 Daily Standup (cada sessão)
  • →Máximo 10 minutos. 3 perguntas por pessoa:
  • ·O que fiz desde a última sessão?
  • ·O que vou fazer hoje?
  • ·Estou bloqueado em alguma coisa?
  • →Atualizar o board (mover cards) logo no início da standup.
  • →Bloqueios resolvidos fora da standup, não durante.
3 Git Workflow obrigatório
  • →Branch por task: feat/BE-01-django-models
  • →Commits com prefixo: feat: fix: test:
  • →PR aberto → task vai para In Review.
  • →Code review de pelo menos 1 pessoa (Adrien revê tudo do Cristian).
  • →CI verde + aprovação → merge → task vai para Done.
  • →Nunca fazer push direto para main.
4 Dependências críticas
  • ⚠BE-01 desbloqueia BE-02 — API só pode ser construída depois dos modelos.
  • ⚠FE-02 desbloqueia CRS-01 — Cristian precisa dos componentes base.
  • ⚠BE-02 desbloqueia FE-03 — Homepage precisa de dados reais da API.
  • ⚠CI-01 deve ser concluído primeiro — todos os PRs dependem do pipeline.
  • ⚠FE-01 e BE-01 podem correr em paralelo — nenhum bloqueia o outro.
5 Mid-Sprint Check (Semana 2)
  • →Reunião de 45min a meio do sprint para avaliar progresso.
  • →Meta mínima: BE-01, BE-02 e FE-01 em Done ou Review.
  • →Se BE-01 ainda não está concluído: Adrien + Diogo ajudam-se mutuamente — é o bloqueador maior.
  • →Rever carga de trabalho do Cristian — ajustar se necessário.
  • →Identificar o que pode escalar para o Sprint 2 se necessário.
6 Sprint Review + Retrospetiva (Semana 3)
  • →Demo ao formador: app a correr end-to-end em dev (make start-dev).
  • →Mostrar: Admin Django funcional, API em /api/docs/, Homepage ligada à API.
  • →Retrospetiva rápida (15min): O que correu bem? O que mudar? Uma ação concreta para o Sprint 2.
  • →Atualizar o backlog do Sprint 2 com o que ficou por fazer.
  • →Git tag sprint1-end no último commit do sprint.
